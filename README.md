# CS360

The app I developed for Project 3 was designed to help users manage and organize inventory while also offering an SMS feature for quick communication or notifications. The goal of the app was to make tracking items—such as quantity, location, and name, simple and efficient for small business owners or individuals managing personal stock. This app was built around user needs for convenience, clarity, and accessibility, ensuring that tasks like adding, viewing, or updating inventory could be done quickly without technical complexity.

To meet these needs, I designed several key screens: a login screen, an inventory management screen, and an SMS manager screen. The login screen helps protect user data while keeping the entry process straightforward. The inventory screen allows users to add, view, and update items with clear input fields and an organized list layout. Finally, the SMS manager lets users send text messages directly from the app, offering flexibility for users who may want to alert others about inventory changes or low stock. The UI was designed with simplicity and usability in mind, using clear labels, proper spacing, and consistent navigation. I focused on ensuring that each screen had one clear purpose so users would not feel overwhelmed or confused. This approach made the interface intuitive and user-centered, allowing smooth transitions between tasks.

When coding the app, I took a structured and modular approach. I started by building the DatabaseHelper class to manage all local data storage using SQLite, ensuring data consistency and reliability. Then, I developed separate activities for each major function, which kept the code organized and easier to debug. I also used object-oriented design principles by creating separate classes, such as InventoryItem and InventoryAdapter, to handle data and display logic. Throughout development, I relied on incremental testing, running the app after each small feature was added—to catch issues early. This strategy helped maintain stability as new code was introduced and made it easier to pinpoint errors.

Testing was a major part of my process. I verified that all buttons, forms, and database operations worked as intended, especially when dealing with edge cases like empty input fields or invalid quantities. I also checked that the SMS feature worked correctly with Android’s permissions system. This testing was critical not only to confirm functionality but also to ensure the app handled real-world scenarios gracefully without crashing. It revealed small logic errors early and helped refine the user experience to be smoother and more reliable.

Throughout the design and development process, one of my biggest challenges was managing permissions for sending SMS messages and storing data locally. I had to research Android’s security policies and make sure the manifest included only the necessary permissions. Overcoming this required careful reading of Android documentation and multiple test runs, which deepened my understanding of secure app design.

The component I was most successful with was the inventory management system. It effectively integrates database operations, dynamic UI updates through RecyclerView, and clean data handling. Building this part of the app helped me demonstrate my growing skills in Android development, including database integration, UI design, and logical problem-solving. Overall, this project helped me connect both the technical and design sides of app development, giving me a stronger foundation for creating user-centered, fully functional mobile applications in the future.
